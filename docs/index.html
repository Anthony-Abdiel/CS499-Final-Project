Biomedical Image Classification & Counting

Robust Cell Image Classification and Counting

A comparison of a Lightweight CNN, ResNet-18, and Vision Transformer on biomedical image classification and counting tasks, with additional testing under blur and noise.

View Results View Repository

Project Overview

What this project is about

This project explores how different deep learning models perform on biomedical image analysis tasks. The first task focuses on classifying blood cell images, while the second task focuses on estimating the number of cells in microscopy images.

Rather than only looking at performance on clean images, this project also tests how the models behave when the images are degraded with blur and noise. This was important because real biomedical images are not always perfect. They may be blurry, noisy, out of focus, or affected by differences in the image collection process.

The overall goal was to compare model accuracy, counting performance, and robustness in a way that gives a more realistic view of each model's strengths and weaknesses.

Research Question

How reliable are these models when image quality changes?

How do CNN-based models and transformer-based models compare on biomedical classification and counting tasks, especially when the input images are affected by blur or noise?

Datasets

Biomedical image datasets used

BloodMNIST

BloodMNIST was used for the classification portion of the project. The goal was to train each model to classify small blood cell images into the correct cell category.

  • Task: image classification
  • Data type: blood cell images
  • Metric focus: test accuracy and validation accuracy
  • Robustness tests: Gaussian blur and Gaussian noise

BBBC005

BBBC005 was used for the counting portion of the project. Instead of predicting a class label, the models had to estimate the number of cells in each image.

  • Task: cell counting / regression
  • Data type: synthetic microscopy images
  • Metric focus: MAE and RMSE
  • Robustness tests: built-in blur levels and Gaussian noise

Models

Architectures compared

Lightweight CNN

The Lightweight CNN was the simplest model in the comparison. It served as a practical baseline to see how far a smaller convolutional model could go without the complexity of larger pretrained architectures.

ResNet-18

ResNet-18 was included as a strong CNN-based model. Its residual connections allow it to learn deeper feature representations, making it a useful comparison point against both the Lightweight CNN and the Vision Transformer.

Vision Transformer

The Vision Transformer uses image patches and self-attention instead of relying only on convolutional filters. This made it useful for comparing a transformer-based image model against more traditional CNN approaches.

Method

Experimental approach

1

Train each model on the normal training data for the selected task.

2

Evaluate each trained model on clean test images.

3

Apply blur and noise corruptions to test how performance changes.

4

Compare the models across accuracy, error, robustness, and practicality.

Results

Classification results on BloodMNIST

Clean Test Accuracy

On the clean BloodMNIST test set, all three models performed strongly. ViT-B/16 and ResNet-18 were very close near the top, while the Lightweight CNN was slightly lower but still competitive. This shows that even the simpler CNN learned useful features for the classification task.

Blur Robustness

The blur test shows a clear difference in robustness. ViT-B/16 and ResNet-18 remained highly stable as blur increased, while the Lightweight CNN dropped more noticeably. This suggests that the larger models were better at preserving classification performance when image detail became less sharp.

Noise Robustness

Noise affected the models more differently than blur. ViT-B/16 stayed strong across all tested noise levels, while ResNet-18 dropped sharply at the highest noise level. The Lightweight CNN also declined, but its drop was more gradual.

Validation Accuracy

The validation accuracy curves show that ResNet-18 and ViT-B/16 reached high accuracy fairly quickly. The Lightweight CNN improved more gradually, but still reached a solid level of performance by the end of training.

Validation Loss

The validation loss curves give another view of the training behavior. ViT-B/16 generally maintained low validation loss, while ResNet-18 showed more variation from epoch to epoch. The Lightweight CNN improved steadily, though it did not reach the same low-loss region as consistently.

BloodMNIST Corruption Examples

These examples show how the BloodMNIST images change under the corruption settings. The goal was to test the models under conditions that reduce visual quality while still preserving the general structure of the input image.

Results

Counting results on BBBC005

Clean Test MAE

On clean BBBC005 images, ViT-B/16 achieved the lowest mean absolute error, followed by ResNet-18 and then the Lightweight CNN. The differences were not extreme, which suggests that all three models learned the counting task reasonably well under normal image conditions.

Blur Robustness: MAE

Under increasing blur, ResNet-18 became much less reliable for counting. Its MAE rose quickly as blur increased. The Lightweight CNN changed more slowly, and ViT-B/16 stayed in the middle, showing better stability than ResNet-18 but not always beating the Lightweight CNN.

Blur Robustness: RMSE

The RMSE results tell a similar story. ResNet-18 became increasingly sensitive as blur level increased, meaning its larger errors became more severe. The Lightweight CNN was surprisingly stable, especially compared with ResNet-18.

Noise Robustness: MAE

Gaussian noise had a major effect on the counting task. ResNet-18 was especially sensitive, with error increasing dramatically once noise was introduced. ViT-B/16 handled noise much better, while the Lightweight CNN also remained more usable than ResNet-18 at higher noise levels.

Noise Robustness: RMSE

The RMSE plot makes the severity of the noise effect even clearer. ResNet-18 produced much larger errors under noise, while ViT-B/16 stayed comparatively stable. This supports the conclusion that ViT-B/16 was the most robust model in the counting experiments.

Validation Loss

The counting validation loss decreased heavily during early training, especially for the larger models. Later epochs show smaller improvements and some instability.

Validation MAE

Validation MAE improved quickly at first, then leveled off. ViT-B/16 eventually became the most consistent, while the Lightweight CNN had occasional spikes.

Validation RMSE

The RMSE curve highlights larger counting mistakes. The Lightweight CNN had more visible spikes, while ViT-B/16 settled into a more stable range later in training.

BBBC005 Corruption Examples

These examples show how blur and noise change the microscopy images used in the counting task. Since counting depends heavily on object visibility, these corruptions can make the regression problem much harder than the clean-image version.

Conclusions

What the results suggest

ViT-B/16 was the strongest overall

ViT-B/16 performed strongly in both the classification and counting experiments. It also showed strong robustness against blur and noise, especially compared with the other models in the counting task.

This was one of the more interesting results because the project used limited biomedical data. Even with that constraint, the Vision Transformer still provided strong performance and stability.

The Lightweight CNN stayed competitive

The Lightweight CNN was not always the best model, but it held up well in many parts of the project. In some robustness tests, especially counting under blur, it was more stable than expected.

This suggests that simpler architectures can still provide real value, especially in situations where compute resources, training time, or deployment complexity matter.

ResNet-18 was strong but fragile

ResNet-18 performed well under clean conditions and was a strong model overall. However, it showed severe sensitivity to image corruption, especially in the counting experiments.

The noise results in particular suggest that strong clean-image performance does not always translate into robust performance when image quality gets worse.

Discussion

Key takeaways

Clean performance is only part of the story

A model can look very strong on normal test data but still struggle when the input images are degraded. The corruption experiments helped reveal differences that clean test accuracy alone would not show.

Counting was more sensitive than classification

The counting task reacted more dramatically to corruption than the classification task. This makes sense because counting depends heavily on local object visibility, and blur or noise can directly interfere with that.

Model complexity is not automatically better

Larger models can be powerful, but the Lightweight CNN showed that simpler models can still be useful. For limited-resource settings, a smaller model may be easier to train, run, and deploy.

Robustness should be part of model evaluation

For biomedical imaging, it is not enough to ask which model performs best in ideal conditions. A useful model should also be tested under conditions that resemble real-world image quality problems.

Limitations

Important constraints

  • Biomedical datasets can be limited in size compared to general image datasets.
  • Some models required image resizing, which may affect fine image details.
  • BBBC005 is synthetic, so performance may not fully represent real microscopy data.
  • Artificial blur and noise do not cover every possible real-world imaging problem.
  • Pretrained models may not transfer perfectly from natural images to biomedical images.
  • Only a small set of model architectures was tested, so the results are not exhaustive.
